03 / Agent

One agent, four verification loops, four human touchpoints

The pipeline is an offline CLI (research.py) — every stage below is a file in the repo, and every number on this page is generated by it.

Stage 1 · composio_lookup.py Composio catalog check

Does a Composio toolkit already exist? SDK when available, public-catalog HTTP fallback otherwise. Deterministic — the LLM never touches this field.

Stage 2 · docs_research.py Search + fetch evidence

Web search plus developer-subdomain probing, and a dedicated "official MCP server" probe — API reference pages rarely mention MCP, so it gets its own pass.

Stage 3 · synthesis.py Synthesize the locked schema

An LLM fills the 19-field record and may cite ONLY fetched, resolving URLs. Thin evidence caps confidence. Reasoning logged per app in out/reasoning/.

Stage 4 · verify.py + browser_verify.py Independent re-checks

Blind re-search (fresh query, stored URLs excluded) plus a Browser Use Cloud agent navigating live docs — channels independent of pass 1.

Stage 5 · handcheck.py + corrections.py Human truth, folded back

A human verifies 17 apps against real docs; disagreements from every loop are adjudicated and folded via a documented, re-runnable corrections migration.

Where a human was needed

  • Preseed priors (~20 apps): human hypotheses like "Amazon SP-API is gated" — the pipeline re-confirms each against fresh evidence and trusts the evidence over the prior.
  • Hand-check (17 apps): field-level verification of api_type + auth + access against official docs; misses are published, not hidden.
  • Adjudicating loop disagreements: the browser agent over-called WhatsApp and Pinterest as Self-Serve; a human kept them Gated because production access needs review/verification.
  • MCP evidence sweep: review passes found 30 official MCP servers that the first batch had marked "None"; a false-positive pass also downgraded 3 unsupported "Official" claims — all fixed in corrections.py and guarded by the new MCP probe.

What defeated the agent (honesty)

  • Paygent Connect: first pass researched the wrong product (an LLM-cost SDK at paygent.to). Marked honestly as unconfirmed, confidence 0.3 — not guessed.
  • Thin or unfetchable docs (iPayX, Pumble, LiveAgent…): logged to failures.log, confidence capped, flagged for the browser loop instead of invented.
  • Free-tier model failures mid-batch: logged, retried through the provider fallback chain — never silently skipped.
  • Composio usage: the SDK/catalog is the authoritative source for the toolkit-overlap column; a read-only tool-call demo lives in demo/composio_demo.py.

Hand-check risky rows and optionally run Browser Use Cloud for live-doc verification.

python research.py --handcheck-template 18
python research.py --fold-handcheck
