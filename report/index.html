03 / Method

Research system and review boundaries

Offline CLI, locked 19-field schema, evidence URLs, and explicit review checkpoints.

Stage 1 · composio_lookup.py Composio coverage

SDK or catalog lookup sets toolkit coverage. The LLM does not decide this field.

Stage 2 · docs_research.py Evidence collection

Ranked official-doc searches reserve fetch slots for auth and production-access evidence.

Stage 3 · synthesis.py Schema synthesis

The LLM cites fetched URLs. Code rejects invalid labels, invented links, and contradictory decisions.

Stage 4 · verify.py + browser_verify.py Independent checks

Fresh-query and live-browser checks record disagreements without changing researched rows.

Stage 5 · handcheck.py Human adjudication

Official docs decide disputed fields; current accuracy stays separate from historical snapshots.

Review boundaries

  • Preseed: a hypothesis narrows the search; current evidence wins.
  • 27 hand-checks: API type, auth, and access are checked against official docs.
  • Disagreements: production access is adjudicated before any model output becomes a correction.
  • MCP audit: first-party MCP pages are checked separately from normal API documentation.

Known limits

  • Paygent Connect: the docs pointed to a different product, so the row remains unconfirmed.
  • Thin docs: iPayX, Pumble, and LiveAgent are capped for confidence and flagged for review.
  • Provider failures: failed model calls are retried through the configured fallback chain.
  • Coverage source: Composio SDK/catalog lookup owns the toolkit-overlap field.
